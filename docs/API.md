## 1. Overview

Although ProcessGuard is a desktop application with no external HTTP endpoints, its modules communicate through well-defined internal Java APIs. This document covers every public method across all 22 source classes, organised by package and layer.

**Package structure:**

| Package | Contents |
|---|---|
| `com.processguard` | `Launcher`, `ProcessGuardMain` |
| `com.processguard.core` | `AlertEngine`, `AppConfig`, `ConfigState`, `CustomRuleEngine`, `HistoryStorage`, `ProcessKiller`, `ProcessMonitor`, `ProcessScanner`, `ReportExporter`, `RuleActionExecutor`, `RuleEvaluator` |
| `com.processguard.listeners` | `AlertListener`, `ProcessListener` |
| `com.processguard.models` | `AlertEvent`, `AlertType`, `Condition`, `CustomRule`, `ProcessInfo`, `RuleAction`, `Severity`, `Status` |
| `com.processguard.ui` | `AlertSidebarManager`, `MainDashboard`, `ProcessTableManager`, `RuleManagerDialog`, `StatusBarManager`, `ToolbarManager` |

## 2. Listener Interfaces

### 2.1 `ProcessListener`

**Package:** `com.processguard.listeners`  
**Description:** Observer interface implemented by any component that needs to respond to process monitoring events. `ProcessMonitor` is the subject; `AlertEngine`, `CustomRuleEngine`, and `MainDashboard` are the registered observers.

---

#### `onNewProcesses(List<ProcessInfo> newProcesses)`

| Attribute | Detail |
|---|---|
| **Description** | Called when one or more processes have started since the previous scan cycle. |
| **Parameters** | `newProcesses` — non-null list of newly detected `ProcessInfo` objects (may be empty if no new processes started) |
| **Returns** | `void` |
| **Called by** | `ProcessMonitor.notifyListeners()` on the `ProcessGuard-Monitor` thread |

**Example implementation:**
```java
@Override
public void onNewProcesses(List<ProcessInfo> newProcesses) {
    for (ProcessInfo process : newProcesses) {
        System.out.println("New process: " + process.getName());
    }
}
```

---

#### `onExitedProcesses(List<ProcessInfo> exitedProcesses)`

| Attribute | Detail |
|---|---|
| **Description** | Called when one or more processes have exited since the previous scan cycle. |
| **Parameters** | `exitedProcesses` — non-null list of `ProcessInfo` objects representing processes that are no longer running |
| **Returns** | `void` |
| **Called by** | `ProcessMonitor.notifyListeners()` on the `ProcessGuard-Monitor` thread |

**Example implementation:**
```java
@Override
public void onExitedProcesses(List<ProcessInfo> exitedProcesses) {
    exitedProcesses.forEach(p -> System.out.println("Exited: " + p.getName()));
}
```

---

#### `onSnapshotUpdate(List<ProcessInfo> currentSnapshot)`

| Attribute | Detail |
|---|---|
| **Description** | Called with the complete list of all running processes after every scan cycle. This is the primary event for bulk evaluation. |
| **Parameters** | `currentSnapshot` — non-null list of all currently running `ProcessInfo` objects, already classified |
| **Returns** | `void` |
| **Called by** | `ProcessMonitor.notifyListeners()` on the `ProcessGuard-Monitor` thread |

---

### 2.2 `AlertListener`

**Package:** `com.processguard.listeners`  
**Description:** Observer interface implemented by any component that needs to display or react to alert events. `AlertEngine` and `RuleActionExecutor` are the subjects; `MainDashboard` is the registered observer.

---

#### `onAlert(AlertEvent alert)`

| Attribute | Detail |
|---|---|
| **Description** | Called when a new alert is generated by the alert engine or a rule action. |
| **Parameters** | `alert` — non-null `AlertEvent` containing PID, severity, type, message, and timestamp |
| **Returns** | `void` |
| **Called by** | `AlertEngine.notifyAlertListeners()` and `RuleActionExecutor.notifyAlertListeners()` |

**Example implementation:**
```java
@Override
public void onAlert(AlertEvent alert) {
    Platform.runLater(() -> alertSidebarManager.addAlert(alert));
}
```

## 3. Service Layer APIs

### 3.1 `ProcessScanner`

**Package:** `com.processguard.core`  
**Description:** Stateless (for calling code), thread-safe class that enumerates all running OS processes and returns classified `ProcessInfo` snapshots. Internally maintains CPU-delta state between calls.

---

#### `scanProcesses()`

| Attribute | Detail |
|---|---|
| **Description** | Enumerates all running processes, collects CPU and memory metrics, resolves process names, and classifies each process. |
| **Parameters** | None |
| **Returns** | `List<ProcessInfo>` — snapshot of all running processes; never null, may be empty |
| **Throws** | Does not throw; individual process failures are silently swallowed |
| **Thread safety** | Safe to call concurrently; internal `ConcurrentHashMap` fields guard CPU-delta state |
| **Side effects** | Updates `previousCpuMillis` and `lastScanTimeMs` internal state; updates `psNameCache` |

**Example usage:**
```java
ProcessScanner scanner = new ProcessScanner();
List<ProcessInfo> processes = scanner.scanProcesses();
processes.forEach(p -> System.out.println(p.getName() + " " + p.getCpuUsage()));
```

---

#### `getOwnPid()`

| Attribute | Detail |
|---|---|
| **Description** | Returns the PID of the currently running Java process (ProcessGuard itself). |
| **Parameters** | None |
| **Returns** | `long` — PID of the current JVM process |

**Example usage:**
```java
long selfPid = scanner.getOwnPid();
System.out.println("ProcessGuard PID: " + selfPid);
```

### 3.4 `ProcessKiller`

**Package:** `com.processguard.core`  
**Description:** Stateless utility for force-killing OS processes by PID. All methods are static.

---

#### `kill(long pid)`

| Attribute | Detail |
|---|---|
| **Description** | Force-terminates a process by PID using OS-specific commands. |
| **Parameters** | `pid` — process ID to terminate; must be > 0 |
| **Returns** | `boolean` — `true` if the OS command exited with code 0, `false` on failure or if PID ≤ 0 |
| **Windows** | Executes: `taskkill /PID <pid> /F` |
| **macOS/Linux** | Executes: `kill -9 <pid>` |

**Example usage:**
```java
boolean killed = ProcessKiller.kill(1234L);
if (!killed) {
    System.err.println("Failed to kill PID 1234");
}
```

### 3.5 `ReportExporter`

**Package:** `com.processguard.core`  
**Description:** Generates PDF summary reports using raw PDF 1.4 syntax. All methods are static.

---

#### `export(List<ProcessInfo> processes, List<AlertEvent> alerts)`

| Attribute | Detail |
|---|---|
| **Description** | Generates a PDF report and saves it to `~/Desktop/ProcessGuardReports/report_<timestamp>.pdf`. |
| **Parameters** | `processes` — current process snapshot (may be empty); `alerts` — alert history (may be empty) |
| **Returns** | `Path` — absolute path to the written PDF file |
| **Throws** | `IOException` — if directory creation or file writing fails |
| **Side effects** | Creates `~/Desktop/ProcessGuardReports/` if it does not exist; writes a `.pdf` file |

**Example usage:**
```java
new Thread(() -> {
    try {
        Path path = ReportExporter.export(snapshot, alerts);
        System.out.println("Report saved to: " + path);
    } catch (IOException e) {
        System.err.println("Export failed: " + e.getMessage());
    }
}).start();
```

## 4. Core Layer APIs

### 4.1 `ProcessMonitor`

**Package:** `com.processguard.core`  
**Description:** Orchestrates scheduled process scanning, detects new and exited processes, and notifies observers.

---

#### `ProcessMonitor(HistoryStorage historyStorage)`

| Attribute | Detail |
|---|---|
| **Description** | Constructs the monitor with a history storage reference. Creates an internal `ProcessScanner`. |
| **Parameters** | `historyStorage` — used to persist snapshots; may be null (disables snapshot persistence) |

---

#### `addListener(ProcessListener listener)`

| Attribute | Detail |
|---|---|
| **Description** | Registers an observer to receive process lifecycle events. Thread-safe (backed by `CopyOnWriteArrayList`). |
| **Parameters** | `listener` — observer to register; ignored if null |
| **Returns** | `void` |

**Example usage:**
```java
processMonitor.addListener(alertEngine);
processMonitor.addListener(customRuleEngine);
processMonitor.addListener(mainDashboard);
```

---

#### `removeListener(ProcessListener listener)`

| Attribute | Detail |
|---|---|
| **Description** | Unregisters a previously registered observer. |
| **Parameters** | `listener` — observer to remove |
| **Returns** | `void` |

---

#### `start()`

| Attribute | Detail |
|---|---|
| **Description** | Starts the periodic scan scheduler if not already running. Uses `AtomicBoolean.compareAndSet(false, true)` to prevent double-starts. The scan thread is a daemon named `ProcessGuard-Monitor`. |
| **Returns** | `void` |
| **Side effects** | Creates `ScheduledExecutorService`; first scan fires immediately (initial delay = 0) |

---

#### `stop()`

| Attribute | Detail |
|---|---|
| **Description** | Stops the periodic scan scheduler. Waits up to 5 seconds for a graceful shutdown, then calls `shutdownNow()`. Uses `AtomicBoolean.compareAndSet(true, false)` to prevent double-stops. |
| **Returns** | `void` |

---

#### `scanNow()`

| Attribute | Detail |
|---|---|
| **Description** | Performs an immediate scan on a new background thread (`ProcessGuard-ManualScan`). Safe to call from the JavaFX Application Thread. |
| **Returns** | `void` |

---

#### `isRunning()`

| Attribute | Detail |
|---|---|
| **Description** | Returns whether the scan scheduler is currently active. |
| **Returns** | `boolean` |

---

#### `getCurrentProcesses()`

| Attribute | Detail |
|---|---|
| **Description** | Returns a snapshot of the most recently scanned processes from the internal `lastSnapshot` map. |
| **Returns** | `List<ProcessInfo>` — defensive copy |

---

#### `getProcessCount()`

| Attribute | Detail |
|---|---|
| **Description** | Returns the number of processes in the last snapshot. |
| **Returns** | `int` |

## 5. Model APIs

### 5.1 `ProcessInfo`

**Package:** `com.processguard.models`  
**Description:** Mutable-status snapshot of a running OS process. Equality is determined by PID alone.

---

| Parameter | Type | Description |
|---|---|---|
| `pid` | `long` | OS process ID |
| `name` | `String` | Process executable name; defaults to `"unknown"` if null |
| `executablePath` | `String` | Full path to executable; defaults to `""` if null |
| `cpuUsage` | `double` | CPU usage %; clamped to minimum 0.0 |
| `memoryUsageMB` | `long` | Memory usage in MB; clamped to minimum 0 |
| `parentPid` | `long` | Parent PID; -1 if unknown |
| `startTime` | `Instant` | Process start time; defaults to `Instant.EPOCH` if null |

| `getMemoryUsageMB()` | `long` | Memory usage in MB |
| `getParentPid()` | `long` | Parent PID |

| `getCapturedAt()` | `Instant` | Time this snapshot was taken |

| `getFlagReason()` | `String` | Flag reason or null |

| `unflag()` | `void` | Removes flag |

### 5.4 `Condition`

**Package:** `com.processguard.models`  
**Description:** A single condition within a `CustomRule`. Field, operator, and value are trimmed on construction; operator is upper-cased.

---

#### Constructor: `Condition(String field, String operator, String value)`

| Parameter | Type | Allowed Values |
|---|---|---|
| `field` | `String` | `"name"`, `"executablePath"`, `"cpuUsage"`, `"memoryUsageMB"`, `"pid"`, `"parentPid"` |
| `operator` | `String` | `"EQUALS"`, `"CONTAINS"` (string fields); `"GREATER_THAN"`, `"LESS_THAN"`, `"EQUALS"` (numeric fields) |
| `value` | `String` | Target value (strings are case-insensitive; numerics parsed with `Double.parseDouble` / `Long.parseLong`) |

---

#### Key Methods

| Method | Returns | Description |
|---|---|---|
| `getField()` | `String` | Condition field name |
| `getOperator()` | `String` | Comparison operator |
| `getValue()` | `String` | Comparison value |

**Example usage:**
```java
Condition condition = new Condition("cpuUsage", "GREATER_THAN", "80");
// Matches processes where cpuUsage > 80.0
```

## 7. Enum Reference

### 7.1 `Status`

| Value | Meaning |
|---|---|
| `NORMAL` | Process is safe and within thresholds |
| `SUSPICIOUS` | Process exceeds CPU or memory threshold |
| `BLOCKED` | Process matches the blacklist |
| `WHITELISTED` | Process is explicitly trusted |

### 7.2 `Severity`

| Value | UI Colour | Usage |
|---|---|---|
| `LOW` | Blue | Unknown/suspicious processes |
| `MEDIUM` | Orange | High memory usage alerts |
| `HIGH` | Red | High CPU usage alerts; custom rule default |
| `CRITICAL` | Red + Bold | Blacklisted process detected |

### 7.3 `AlertType`

| Value | Description |
|---|---|
| `HIGH_CPU_USAGE` | Process CPU exceeds threshold |
| `HIGH_MEMORY_USAGE` | Process memory exceeds threshold |
| `BLACKLISTED_PROCESS` | Process is on the blacklist |
| `UNKNOWN_PROCESS` | Process is classified SUSPICIOUS |
| `SUSPICIOUS_PARENT` | Reserved for future parent-chain detection |
| `RAPID_SPAWN` | Reserved for future rapid process spawning detection |
| `CUSTOM_RULE_VIOLATION` | A user-defined rule matched |

### 7.4 `RuleAction`

| Value | Behaviour |
|---|---|
| `LOG_ONLY` | Print to stdout; no alert created |
| `ALERT_ONLY` | Create `AlertEvent`, persist, notify observers |
| `KILL_PROCESS` | Same as ALERT_ONLY + `ProcessKiller.kill(pid)` |

#### `RuleAction.fromString(String value)`

| Parameter | Description |
|---|---|
| `value` | String to parse (e.g., `"KILL_PROCESS"`); case-insensitive |
| **Returns** | `RuleAction` — parsed value, or `ALERT_ONLY` if null or unrecognised |
